Improve Order Detail Line Item Matching

Implementation of AI to streamline invoice processing by analyzing invoices then indicating matches and educated guesses.

The Problem

Users are not taking the time to match returned, unmatched invoice line items to items within the app. If we could eliminate the human factor for more line items, we would get better adoption rates.

The Solution

  • Improve the order detail line item UI so it is more readable
  • Provide indicators when AI successfully matched an item, took a guess and did not attempt to guess
  • Give the user the ability to mark a line item as checked

Improving Line Item UI

By employing the 'Crazy 8s' method, I generated eight distinct mockups using Figma. These mockups explore various ways to showcase order detail line items.

Creating Matching Logic UI

After carefully analyzing the requirements, it was determined that three options would be available: a 100% match, an educated guess, or no guess at all. Working with Product, we wrote out the scenarios for the three situations.

100% Match

  • Invoice item sku matches sku of a packaging/vendor relationship or item and invoice line item has been mapped previously
  • Packaging matches
  • Vendor exists on that given packaging or item

Educated Guess

  • Invoice item sku matches sku of a packaging/vendor relationship or item but invoice line item has ever been mapped before
  • Any assumptions made for qty or price
  • Invoice line item name matches item card name 1:1
  • Packaging not null or matchable to dropdown
  • Last three orders same map to packaging

No Guess At All

  • Packaging is null or un-matchable to dropdown

Once the requirements were finalized, I went to Figma to devise a strategy for effectively notifying the user about the various scenarios in the UI.

  • No color would indicate that we are 100% confident that this line item is correct
  • Yellow/ Orange would be an educated guess, alerting user to double check this with their original order and invoice
  • Red would mean we were unable to make an educated guess or 100% match
  • The user could hover over the packaging and get the actual line item information from the invoice that was extracted
  • The user would be able to check that they looked for the line item and everything is good
  • Once a selection or action was made on the no guess items, they would appear good
  • If a user checked the line item using the checkmark, the educated guess items would show as no action
  • After all line item information is added/ checked by the user, the ideal state would be every line item with green check mark
  • If item does not have a match at all, once matched the UI would show markings


Next Steps
